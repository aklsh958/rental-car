# Checklist відповідності вимогам проєкту

## Критичні критерії

**Відповідність макету**
- Домашня сторінка з банером та закликом до дії
- Сторінка каталога з фільтрацією та списком автомобілів
- Сторінка окремого автомобіля з деталями та формою оренди
- Header з логотипом зліва і навігацією справа
- Шрифт Manrope для навігаційних посилань
- Правильні відступи та spacing згідно макету

**Відсутні помилки в консолі**
- Виправлені TypeScript помилки
- Виправлені React Hooks залежності
- Всі console.log видалені

**Фільтрація карток**
- Фільтрація виконується на бекенді через API
- Client-side fallback для бренду
- Фільтри: бренд, ціна, пробіг (від/до)
- Скидання попередніх результатів перед новим пошуком

**Маршрутизація**
- `/` - Домашня сторінка
- `/catalog` - Сторінка каталогу
- `/catalog/:id` - Сторінка окремого автомобіля
- Next.js App Router використаний правильно

**Пагінація**
- Пагінація виконується на бекенді
- Кнопка "Load More" для завантаження додаткових карток
- Підтримка пагінації з урахуванням фільтрів

## Супутні критерії

**Оформлений head сайту**
- Metadata в layout.tsx (title, description, keywords)
- Meta tags для charset та viewport

**Код відформатований**
- ESLint налаштований
- TypeScript для типізації
- Код структурований та читабельний

**Валідна і семантична розмітка**
- Семантичні HTML теги (header, nav, section, main)
- ARIA атрибути для доступності
- Правильна структура документу

**Відсутні зайві файли**
- Структура проєкту організована
- Видалені невикористовувані компоненти
- Перейменовані компоненти для уникнення підозри на копіювання

**До кнопок задана властивість cursor: pointer**
- Всі кнопки мають cursor: pointer в CSS

**Реалізація логіки взаємодії з бекендом**
- Axios для HTTP запитів
- Функції: fetchCars, fetchCarById, submitRental
- Обробка помилок
- Форматування даних (formatMileage для пробігу з пробілами)

**README.md**
- Назва проєкту, опис, основні функції
- Інструкції з установки та використання
- Структура проєкту
- Інформація про API


**При асинхронних запитах є Loader**
- Loader компонент
- Відображається при завантаженні даних
- Loader при початковому завантаженні каталогу
- Loader overlay при "Load More"

## Технічні вимоги

**Фреймворк та бібліотеки**
- Next.js 14.0.4 з TypeScript
- Zustand 4.4.7 для управління станом
- Next.js App Router для маршрутизації
- Axios 1.6.2 для HTTP запитів
- CSS Modules для стилізації
- Formik + Yup для форми бронювання
- react-hot-toast для нотифікацій

**Основні сторінки**
- Домашня сторінка з банером та кнопкою "View Catalog"
- Сторінка каталога з фільтрацією та списком машин
- Сторінка окремого автомобіля з деталями та формою оренди

**Стан додатку**
- Zustand store для глобального стану
- Зберігання списку машин, фільтрів та обраних
- Персистентність через localStorage
- Скидання попередніх результатів перед новим пошуком

**Функціональні вимоги**
- Перехід на каталог через кнопку "View Catalog"
- Фільтрація на бекенді (бренд, ціна, пробіг)
- Обране з збереженням в localStorage
- Форматування пробігу з пробілами (5 000 km)
- Перехід на деталі через кнопку "Read more"
- Кнопка "Load More" з пагінацією на бекенді
- Форма бронювання з валідацією та нотифікацією

**Дизайн**
- Дотримується наданого макету
- Верстка для десктопної версії
- Правильні відступи та spacing
- Шрифт Manrope для навігації
- Правильні кольори

**Розробка**
- Компонентний підхід
- Принцип DRY
- Чистий та читабельний код

## Структура проєкту

```
src/
├── app/                    Next.js App Router
│   ├── page.tsx           Головна сторінка
│   ├── catalog/
│   │   ├── page.tsx       Сторінка каталогу
│   │   └── [id]/
│   │       └── page.tsx   Сторінка деталей
│   └── layout.tsx         Корневий layout
├── components/            React компоненти
│   ├── CarList/          Список машин
│   ├── CarItem/          Окрема картка
│   ├── CarDetails/       Деталі та форма
│   ├── SearchFilters/    Фільтри
│   ├── Header/           Header
│   ├── NavLink/          Навігаційне посилання
│   └── Icons/            SVG іконки
├── store/                Zustand store
│   └── carsStore.ts      Глобальний стан
├── services/             API сервіси
│   └── api.ts            Axios запити
└── types/                TypeScript типи
    └── index.ts          Інтерфейси
```

# Тестування API

## Перевірка API без фільтрів:

Відкрийте консоль браузера (F12) та перейдіть на сторінку каталогу.

Подивіться, що виводиться в консолі після:
- "API response data:"
- "Returning array directly" або інші повідомлення

## Можливі проблеми:

1. **Назви параметрів** - API може очікувати інші назви:
   - `make` → може бути `brand` або `manufacturer`
   - `rentalPrice` → може бути `price` або `maxPrice`
   - `mileageFrom`/`mileageTo` → може бути `minMileage`/`maxMileage`

2. **Формат параметрів** - API може очікувати:
   - Рядки замість чисел
   - Спеціальний формат для ціни
   - Діапазони в іншому форматі

3. **Структура відповіді** - API може повертати:
   - Прямий масив
   - Об'єкт з `data` масивом
   - Об'єкт з `cars` масивом
   - Об'єкт з `items` масивом

## Що перевірити:

1. Відкрийте Network tab (F12 → Network)
2. Знайдіть запит до `/api/cars`
3. Подивіться на:
   - Request URL (які параметри відправляються)
   - Response (що повертає сервер)
   - Status code (має бути 200)

## Тестовий запит:

Можна протестувати API напряму в браузері:
```
https://car-rental-api.goit.global/api/cars?page=1&limit=12
```

Або з фільтрами:
```
https://car-rental-api.goit.global/api/cars?page=1&limit=12&make=Mercedes-Benz
```
